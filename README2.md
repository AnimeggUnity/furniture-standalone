# 家具競標管理工具 - Chrome 擴充功能

一個專為新北市政府環保局資源回收網家具競標系統設計的 Chrome 擴充功能，提供數據管理、批量操作、自動化填表和遠端同步等完整功能。

## 📋 功能架構概覽

### 🎯 核心檔案結構
```
.
├── manifest.json          # 擴充功能清單檔，定義所有腳本和權限
├── inject.js              # 注入頁面的腳本，用於存取 Vue 實例和頁面資料
├── content.js             # 內容腳本主入口，初始化所有模組
├── config/                # 靜態設定檔
│   ├── constants.js     # 應用程式常數
│   └── mappings.js      # 類別 ID 映射
├── core/                  # 核心系統模組
│   ├── errorHandler.js  # 統一錯誤處理
│   └── messageHandler.js# 頁面訊息監聽與分派
├── state/                 # 狀態管理模組
│   ├── appState.js      # 全域應用狀態
│   ├── bidStatus.js     # 競標狀態機
│   └── itemStatus.js    # 商品狀態機
├── ui/                    # UI 相關模組
│   ├── styles.js        # UI 元件樣式和變體
│   ├── buttons.js       # 功能按鈕的注入與管理
│   ├── notifications.js # 統一通知系統
│   └── components/      # 各 UI 元件
│       ├── dataPanel.js # 資料面板
│       ├── deleteModal.js # 刪除確認對話框
│       ├── importModal.js # 匯入對話框 (本地與遠端)
│       ├── photoPreview.js# 圖片預覽
│       ├── settingsPanel.js # 設定面板
│       └── statsModal.js  # 統計資料對話框
└── utils/                 # 工具函數模組
    ├── api.js           # 後端 API 封裝
    ├── csv.js           # CSV 匯出功能
    ├── image.js         # 圖片處理 (Base64轉換等)
    ├── scroll.js        # 安全滾動函數
    └── sheetSync.js     # 聯絡人資料同步模組
```

### 🔧 主要功能模組

#### 1. 資料分析與呈現模組
- **資料面板** - 即時顯示完整競標資料，自動查詢最高出價者
- **階層式統計** - 按年/月兩層級展示建立日期與結束日期統計，日期優先排序，視覺化分隔線 🆕
- **圖片預覽** - 內建圖片預覽視窗，支援多圖片展示

#### 2. 資料匯出模組
- **CSV 匯出** - 基本資料與完整資料兩種格式
- **JSON 匯出** - 支援 DL (分離下載) 和 PACK (打包下載) 模式
- **批次匯出** - 多檔案順序下載，支援 manifest.json 索引檔案
- **列印表格** - 自動查詢競標資料，格式化列印競標結果，區分競標中/已結束狀態
- **列印選擇項目** - 資料面板專屬，列印已選項目的編號、名稱與前兩張照片，緊湊 2 欄佈局（一頁 10 個物件）🆕

#### 3. 資料匯入模組
- **JSON 載入** - 本地 JSON 檔案自動填表
- **直接匯入** - 完全繞過前端，直接呼叫後端 API
- **批次匯入** - 多檔案選擇匯入，支援 manifest.json 索引檔案，可自定義競標天數（1-90天）🆕
- **遠端匯入** - 從遠端伺服器獲取檔案清單並批量處理

#### 4. 圖片處理模組
- **Base64 轉換** - 圖片與 Base64 字串互轉
- **批量上傳** - 並行處理多張圖片上傳
- **智能壓縮** - 自動優化圖片檔案大小

#### 5. 資料管理模組 🆕
- **得標者分組功能** - 一鍵切換得標者分組模式，固定客戶購買記錄一目了然 🆕
- **三層智能排序** - 已得標 > 競標中 > 無競標，自動優先顯示重要項目 🆕
- **逾期未付篩選** - 只顯示「已得標 + 未付 + 結束超過 14 天」的項目 🆕
- **長期未結標篩選** - 篩選競標期 ≥20 天的物件，分「競標中（距截標 >7天）」與「無人競標」兩區塊顯示 🆕
- **刷新截標日功能** - 批次將已選物件的截標日更新為今天 +7 天，需確認後執行 🆕
- **極簡視覺分隔** - 藍/橙/灰三色分隔線，零冗餘設計 🆕
- **多模式選擇機制** - 支援「全選」、「全不選」、「只選無競標」三種快速選擇模式
- **競標狀態智能顯示** - 自動區分「競標中」和「已結束」狀態，並以不同顏色標示
- **批次刪除功能** - 支援多項目批次刪除，含確認對話框與安全機制
- **競標狀態檢查** - 實時查詢最高出價者與競標價格，精確顯示競標資訊

#### 6. 聯絡人同步模組 🆕
- **自動資料同步** - 從 PHP API 自動同步得標者聯絡資料（姓名、電話、Email）
- **記憶體快取** - 本地快取聯絡人資料庫，減少 API 請求次數
- **雙重索引** - 支援帳號或 Email 快速查詢聯絡人
- **即時狀態顯示** - 同步按鈕顯示即時狀態（⏳同步中 / ✓已同步 / 🔄未同步）
- **聯絡方式整合** - 資料面板自動顯示得標者聯絡方式（👤姓名、📞電話、📧Email）
- **手動資料優先** - 優先使用手動補充的電話資料，自動資料作為備援
- **API 設定管理** - 支援自訂聯絡人 API URL 與 API Key，含連線測試功能

#### 7. 表單自動化模組
- **Element UI 相容** - 完美支援下拉選單、日期選擇器等複雜組件
- **Vue 響應式** - 確保所有表單更新觸發 Vue 的響應式系統
- **欄位映射** - 自動識別並對應 JSON 資料到正確表單欄位

## 🚀 詳細功能說明

程式碼已重構為模組化結構，各檔案職責分明：

### 模組功能細分

#### `config/` - 設定模組
- **`constants.js`**: 定義所有應用程式級別的常數，如時間、尺寸、顏色等。
- **`mappings.js`**: 管理類別名稱與 ID 之間的映射。

#### `core/` - 核心系統
- **`errorHandler.js`**: 提供統一的 `ERROR_HANDLER`，集中處理所有錯誤，並顯示用戶友好的通知。
- **`messageHandler.js`**: 設置 `window.addEventListener`，監聽來自 `inject.js` 的訊息 (`vue-stats`, `vue-panel-data`, `vue-print` 等)，並分派給對應的處理函數。包含列印表格功能與載入提示。

#### `state/` - 狀態管理
- **`appState.js`**: 初始化 `FurnitureHelper` 命名空間，管理全域狀態。
- **`bidStatus.js`**: `BID_STATUS_SYSTEM` 狀態機，根據競標數據和結束日期判斷狀態（競標中/已結束/無競標），自動套用對應樣式和文字。🆕
- **`itemStatus.js`**: `ITEM_STATUS_SYSTEM` 狀態機，根據付款和取貨狀態決定樣式和文字。

#### `ui/` - 使用者介面
- **`styles.js`**: 定義 `UI_COMPONENTS` 和 `COMPONENT_VARIANTS`，提供 `applyComponentVariant` 函數來動態生成樣式。
- **`notifications.js`**: `showNotification` 函數，提供統一的通知外觀。
- **`buttons.js`**: `insertButtons` 函數，負責將所有功能按鈕注入到頁面中。
- **`components/`**:
    - **`dataPanel.js`**: `buildPanel` 函數，建立側邊資料面板，包含搜尋、得標者分組、聯絡人同步、三模式選擇按鈕（全選/全不選/只選無競標）、批次操作等功能。自動顯示得標者聯絡方式（👤姓名、📞電話、📧Email）。支援長期未結標篩選與批次刷新截標日。🆕
    - **`statsModal.js`**: `showHierarchicalModal` 函數，建立階層式統計資料的互動式樹狀圖（年/月兩層級，日期優先排序，延遲載入）。🆕
    - **`importModal.js`**: 處理本地和遠端匯入的 UI 和邏輯。
    - **`deleteModal.js`**: `showDeleteConfirmationModal` 函數，提供批次刪除前的安全確認。
    - **`photoPreview.js`**: `showPhotoPreview` 函數，顯示商品圖片預覽。
    - **`settingsPanel.js`**: `showSettingsPanel` 函數，提供 Webhook 設定與聯絡人 API 設定的 UI，支援連線測試功能。

#### `utils/` - 工具函數
- **`api.js`**: 封裝 `uploadImage`, `directSubmitToAPI`, `updateProductEndDate` 等與後端 API 的所有互動。
- **`image.js`**: 提供 `convertImageToBase64`, `base64ToFile` 等圖片處理工具。
- **`csv.js`**: `exportToCSV` 和 `exportAllToCSV` 匯出功能。
- **`scroll.js`**: `safeScrollIntoView` 安全滾動函數。
- **`sheetSync.js`**: 聯絡人資料同步模組，提供 `syncContacts`, `getContact`, `updateContact` 等功能。從 PHP API 同步得標者聯絡資料，支援雙重索引（帳號 + Email）快速查詢，優先使用手動補充資料。

#### inject.js 核心功能:

**1. Vue 資料存取**
- 自動搜尋頁面中的 Vue 表格組件
- 透過 `__vue__` 屬性存取 `tableFullData`
- 安全的父組件遍歷機制

**2. 競標記錄查詢**
- `fetchBidLog` - 非同步查詢競標紀錄 API
- 支援各種 UUID 格式識別
- 自動解析最高出價者與價格
- 支援多種出價者欄位名稱 fallback 機制

**3. 階層式資料統計**
- 按 CreateDate 和 EndDate 分別統計
- 三層級結構：年 → 月 → 日 → 商品列表
- 並行處理所有資料以提升效能

**4. 資料面板處理**
- 依序查詢每筆競標記錄
- 標記查詢狀態 (`BidChecked`, `HasBids`)
- 整合競標者資訊與價格資料

**5. 列印表格處理** 🆕
- 自動查詢所有商品的競標資料
- 發送載入中提示訊息
- 完整資料查詢後傳遞給列印功能

- `Length` - 長度 (公分)

## 🛠️ 使用方式

### 安裝步驟
1. 下載擴充功能檔案到本地資料夾
2. 開啟 Chrome 瀏覽器
3. 進入 `chrome://extensions/`
4. 開啟右上角的「開發人員模式」
5. 點擊「載入未封裝項目」
6. 選擇擴充功能資料夾

### 基本操作流程

#### 1. 資料查看與分析
1. 進入新北市再生家具競標頁面
2. 等待頁面完全載入後，按鈕會自動注入
3. 點擊「年度統計」查看階層式統計資料
4. 點擊「資料面板」開啟側邊面板查看詳細資訊

#### 2. 資料匯出
- **匯出** - 基本欄位 CSV 檔案
- **完整匯出** - 所有欄位 CSV 檔案
- **DL** - 分離下載 JSON 和圖片檔案
- **PACK** - 打包下載含 Base64 圖片的完整 JSON
- **列印表格** - 自動查詢競標資料並格式化列印，支援競標狀態區分 🆕

#### 3. 資料匯入
- **載入表單** - 選擇 JSON 檔案自動填表
- **直接匯入** - 選擇含 Base64 圖片的 JSON 直接送出 API
- **遠端匯入** - 從遠端伺服器選擇檔案批量匯入

#### 4. 設定管理
- 點擊「設定」按鈕可配置遠端匯入的 Webhook 網址與聯絡人 API 設定
- 支援連線測試確保設定正確
- 聯絡人 API 支援自訂 URL 與 API Key

#### 5. 聯絡人同步 🆕
- **自動同步**: 開啟資料面板時自動同步聯絡人資料
- **手動同步**: 點擊 🔄 按鈕手動重新同步
- **狀態顯示**: 同步按鈕即時顯示狀態（⏳同步中 / ✓已同步 / 🔄未同步）
- **聯絡方式**: 得標者資訊下方自動顯示 👤姓名、📞電話、📧Email
- **API 設定**: 設定面板可自訂聯絡人 API URL 與 API Key

### 高級功能

#### 階層式統計使用
1. **雙模式切換**：支援「詳細樹狀」和「季報摘要」兩種顯示模式
   - **詳細樹狀模式**：完整的年月日三層展開結構
   - **季報摘要模式**：按月份統計總覽，以網格形式呈現
2. 點擊統計區塊標題可展開/收合整個區塊
3. 點擊年份可展開該年度所有月份
4. 點擊月份可展開該月所有日期
5. 點擊日期可展開該日所有商品詳細資訊
6. 商品資訊包含競標狀態、付款取貨狀態等

#### 資料面板智能選擇 🆕
1. **可拉寬 Panel** 🆕:
   - **拖曳調整**: 將滑鼠移到 panel 左邊緣，游標變為 `↔` 時按住拖曳
   - **寬度範圍**: 350px (最小) ~ 800px (最大)，預設 450px
   - **記憶功能**: 自動儲存偏好寬度，下次開啟維持設定
   - **快速重置**: 雙擊左邊框重置為預設 450px
   - **滾動友善**: 拖曳邊框固定在視窗，panel 滾動時仍可調整
2. **付款取貨狀態顯示** 🆕:
   - **已付/已取** - 綠色文字，訂單完成
   - **已付/未取** - 橙色文字，待取貨
   - **未付/已取** - 紅色文字，異常狀態
   - **未付/未取** - 灰色文字，初始狀態
   - 狀態顯示在追蹤數後、競標狀態前，邏輯分組清晰
3. **五種快速操作**：
    - **全選** - 勾選所有項目，用於批次匯出全部資料
    - **全不選** - 清除所有勾選，重置選擇狀態
    - **只選無競標** - 僅勾選無競標的項目，方便批次處理未出價商品
    - **列印** - 列印已選項目的編號、名稱與前兩張照片（緊湊 2 欄佈局，一頁 10 個物件）
